# Fulcrum-Template-Typst

本仓库为 Fulcrum 模板的 Typst 版本，旨在提供一种基于知识条目与术语索引的高效数学知识文档（笔记）撰写标准。

*本文关于 Typst 语法的讲解不一定都对，官方文档写的很好，建议查询。大模型对 Typst 普遍不熟悉，能写但幻觉比较严重。*

## 使用方法

1. 在文档开始处引入模板：

    ```Typst
    #import "PATH/TO/Fulcrum.typ": *
    ```

2. 设置展示格式：

    依据语言设置文档格式，以中文为例：

    ```Typst
    #show: FulcrumCN
    ```

## 文档

### FulcrumCN 格式说明

目前包含以下设置：

```Typst
  #set heading(numbering: "1.1.")
  #set text(font: ("New Computer Modern", "SimSun"), size: 11pt)
  #set par(first-line-indent: 2em)
  #set enum(indent: 2em)
  #set list(indent: 2em)
  #set figure(supplement: [图])
  #show heading: ApplyBold
  #show strong: ApplyBold
  #show link: set text(weight: "regular", fill: rgb("#000080"))
```

1. `=` 号语法对应的章节编号格式为 `1.1.`，即章节二级编号，各有一个点结尾。
2. 字体以 `New Computer Modern` 为主，辅以 `SimSun`。这么做会自动让拉丁字母套用类 $\LaTeX$ 的 Computer Modern 字体族，而中文则使用宋体，以最大程度符合 $\LaTeX$ 默认格式。
3. 段落首行缩进 2 个汉字宽度，以符合中文排版习惯。Typst 似乎不会对某些情况下的首个段落进行缩进，目前没有研究出怎么解决。
4. 枚举环境和列表环境的缩进设置为 2 个汉字宽度，以符合中文排版习惯。
5. 图表编号设置为“图*”而非“Figure”，以符合中文习惯
6. 章节标题和加粗文本的样式设置为 `ApplyBold`，这一函数会仿照 $\LaTeX$ 对中文字体使用黑体来达到提高字重的效果，这是因为 `SimSun` 字体没有自带的粗体。
7. 带索引的文字颜色设置为深蓝色，以表明索引。

### 函数接口

Typst 是动态类型语言，类型检查要手动写，这部分功能还没有写完。如果你传入了不符合预期类型的参数，此时如果函数内手动写了类型检查则可能相应报错，如果没有写类型检查则报错报在函数声明的位置而非调用位置，导致你搞不清哪里有错，而且点击 Preview 时 LSP 也容易把你带到函数声明的位置，这个设计相当烦人，目前不知道怎么改。

后续会逐渐完善函数的类型检查实现。

Typst 的函数传参方式非常奇怪，分为必填参数（位置参数）和可选参数两种**完全平行**的方式。

* 必填参数 / 位置参数总是没有默认值，传参时必须按顺序填入，不能缺漏。这里的顺序不受可选参数影响。
  
  由于没有默认值，LSP 无法推理必填参数的类型，导致类型显示为 `any`。

* 可选参数总是有默认值，传参时必须用 `参数名 : 值` 语法传入（也就是说不能用位置参数的方法直接传），它的位置、顺序完全无影响。注意它传值用的是 `:` 而非更加常见的 `=` 或 `:=`。

  LSP 似乎会根据默认值的类型和 Sampled Values 中传参的类型来综合推断参数的类型。

* 似乎有办法通过展开列表来传递，但还没搞清楚怎么回事

总结下来的建议是先把必填参数填完过编译，然后可选参数可以以任意顺序插入。LSP 浮窗会给予一定的类型信息提示和 Sampled Values 作为示例，但并没有足够好用。

***（个人认为 Typst 从类型系统到传参方式再到具体符号语法都极蠢，但已经比 $\LaTeX$ 好用很多了只能暂且接受）***

## 吐槽一些 Typst 的语法

Typst 属于半函数式语言，它有很多与 Haskell、Lean 等函数式语言相似的地方，也有很多与传统过程式语言相似的地方，很怪。大部分时候在写函数，但有些地方涉及 `#context` 问题，比如制作定理编号的计数器类似的功能属于有副作用的函数，要小心。这时候写法又比较像 React。

Typst 代码主要分为内容环境、代码环境、公式环境和纯字符串四种。

* 在代码环境中进入内容环境用 `[ ... ]`。

    内容环境里是被渲染的内容，但是里面可以接着嵌套其他环境。

    整个文档最大的作用域默认是内容语境，也就是在一个空文档里直接写字符串将直接被渲染。

* 在内容环境中进入代码环境用 `# ... ;`。

    结尾的分号可以不加，仅在你想推出代码环境但又不想多渲染一个空格时候用。

    进入代码环境后不用再次进入，除非又进入内部嵌套的内容环境。

* 行内公式环境用 `$...$`，行间公式用 `$ ... $`，二者的区别是两个 `$` 内侧是否皆有空格。行内公式也可以使用 display 相关命令按行间方法渲染，但我不记得具体命令了。公式环境内的写法相当怪，属于内容环境和代码环境杂交的状态，需要摸索。说一些重点和 $\LaTeX$ 不一样的地方：

    1. 不用写 `\`，命令可以像代码环境一样直接敲。

    2. 大部分数学符号的名字不一样，要查官方文档，好在比较好查。可以直接输 unicode 字符渲染。一些分支字符如 `$\subseteq$`、`$\pm$` 使用命名空间分支，写作 `$subset.eq$` 和 `$plus.minus$`。

    3. 变量中间必须空格，不然会被当成命令。

    4. 有一些语法糖，比如 `>=` 会渲染成 $\geq$、`$a/b$` 会渲染成 $\frac{a}{b}$，代价是如果想画斜杠反而得在 `frac` 函数里改可选参数。

    5. 可以用 `#show` 命令重载数学符号的显示。

一些很怪的语法：

1. 列表写法是圆括号 `( a, b, c )`，空列表必须写 `(,)`，不加逗号会识别成括号直接被格式化清除。

2. 键值对也是圆括号。空键值对写作 `(:)`。

3. 函数传值是冒号，两种平行传值方式很奇怪。

4. 括号的语法功能太多，嵌套起来乱七八糟，很麻烦，注意观察括号的染色，不同环境下的染色不一样。
